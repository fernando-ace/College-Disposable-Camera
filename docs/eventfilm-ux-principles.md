# EventFilm UX Principles

EventFilm is for casual groups first: college students, friend groups, and small event organizers who need one easy place for event photos.

## Target user

Primary users are college students, friend groups, and small group or event organizers.

Secondary users are small clubs, Greek life groups, teams, and casual event hosts.

Professional event planners and large formal events can still use EventFilm, but they are not the main audience.

## Product promise

EventFilm helps a group collect photos from an event without chasing people afterward.

## Core loop

1. Create an event.
2. Share the link or QR code.
3. Guests upload without an account.
4. The host sees photos come in.
5. Everyone gets a simple shared recap.

## UX principles

- One obvious next action per screen.
- Hide advanced tools until needed.
- Use normal human words.
- Design for phone-first casual use.
- Make empty states explain what to do.
- Make every public page understandable without context.
- Prefer fewer choices over more powerful controls.
- Keep host tools calm and guest pages effortless.
- Never make accountless guests feel like they are using admin software.
- Never make a host wonder which link to send.

## Feature trimming rules

A feature should be visible by default only if it helps one of these jobs:

- Invite people to add photos.
- Add photos.
- See photos coming in.
- Review photos.
- Share the recap.

Everything else should be collapsed, moved to More, or moved to founder/internal views.

## Naming rules

Avoid internal or big-event terms:

- Presenter
- Command center
- Launch signals
- Event signal
- QR rail
- Mode controls
- Recap activity
- Host handoff
- Beta ops

Prefer:

- Photo wall
- Shared album
- Add photos
- Share link
- QR code
- Event recap
- Review photos
- Next steps
