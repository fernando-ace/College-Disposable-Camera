# EventFilm June Revenue Sprint

Goal: get 10 paid events scheduled by June 30, 2026. A paid event means the host has paid the $5 beta price and has an event link or QR code created.

## Operating Rule

Do not build new product surface until 10 paid events are scheduled.

Allowed fixes only:

- Broken QR or event links
- Failed guest uploads
- Failed host download
- Mobile layout that blocks a guest or host from completing the flow
- Production setup issues that prevent a real event from working

Not allowed before the goal:

- Stripe or checkout
- Password reset
- Email verification
- Pricing tiers or plan enforcement
- Admin dashboard
- Custom branding
- Photo editing or album customization
- Analytics beyond a simple manual tracker

## Free Private Beta

Run one privately offered free event within 7 days. Do not advertise this as public pricing; use it as a personal customer-development invite.

Minimum bar:

- At least 10 expected guests
- Host creates or receives an EventFilm event link
- Guests scan the QR code or open the link on phones
- At least 10 real uploads
- Album reveal is tested
- Host can view, delete one test photo, and download the zip

After the event, ask:

- What confused guests?
- Did the host like the album enough to use it again?
- Would the host pay $5 for this as a summer beta event?

Only fix issues that blocked the event flow.

## Summer Offer

Use this offer in DMs and conversations:

> I am running a summer beta for EventFilm. I set up a disposable camera album for your party or event. Guests scan a QR code, upload photos, and you get the full album/download after. It is $5 for the beta.

Payment stays manual for now: Venmo, Cash App, Zelle, cash, or invoice.

Show prospects the website as a demo and credibility page, but keep the main action as texting to book. Say: "Here is what it does. I am personally setting up beta events right now." For hand-picked test hosts, you can privately say: "I am testing this and can set up your first event free."

## Target Buyers

Prioritize easy summer events:

- Birthday parties
- Graduation parties
- Lake or beach trips
- Wedding showers
- Small weddings or receptions
- Club and social gatherings
- Auburn students still in town
- Friends of friends hosting events

## Daily Sales Routine

Every day until June 30:

1. Send 20 direct messages.
2. Ask 3 friends for introductions to people hosting summer events.
3. Post 1 short demo or story showing the QR upload flow.
4. Create event links personally for interested hosts.
5. Record each lead in the manual tracker.

## Outreach Scripts

Warm intro:

> Hey, I am testing EventFilm for summer events. It is a disposable camera album where guests scan a QR code and upload photos, then the host gets the full album after. Do you know anyone hosting a birthday, grad party, lake trip, shower, or social this month?

Direct host message:

> Hey, I am running a private beta for EventFilm for summer events. I can set up a QR code photo album for your event so guests can upload pics without downloading an app. It is $5 for the beta. Want me to make you a sample event link?

Follow-up:

> Quick follow-up. I can set up the EventFilm link and QR for you today, and guests just scan/upload from their phones. Want me to book your event?

After beta:

> Thanks for testing EventFilm. What confused guests, what worked well, and would you pay $5 to use this at another event?

## Managed Booking Flow

When someone is interested:

1. Collect event name, event date, reveal time, and rough guest count by text.
2. Manually create the event link and QR code.
3. Collect $5 manually.
4. Send the host the QR code and event link.
5. If someone creates an event without paying, treat it as a lead and follow up with the $5 managed beta offer.

Future pricing can move toward one-time, capacity-based event pricing after the first 10 paid events. Do not build capacity tiers yet.

## Weekly Checkpoints

Every Sunday:

- Count $5 beta payments.
- Count event links delivered.
- Count real events with uploads.
- List the top 3 objections heard.
- Decide the smallest blocker fix, if any.

If the week produced fewer than 2 paid events, increase outreach before changing the product.
